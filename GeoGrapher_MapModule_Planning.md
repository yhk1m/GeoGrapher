# GeoGrapher 지도 시각화 모듈 기획 문서

## 1. 프로젝트 개요

### 1.1 목적
기존 GeoGrapher(지리 그래프 생성기, geographer.bgnl.kr)에 **지도 기반 시각화 모듈**을 추가한다.
기존 8개 차트 유형(기후, 인구 피라미드, 삼각, 누적 막대/원, 산점도/버블, 하이서그래프, 직육면체, 방사형)에 이어 **3가지 지도 시각화 유형**을 통합하여, 지리 수업에서 필요한 거의 모든 시각화를 하나의 도구에서 제작할 수 있게 한다.

### 1.2 추가되는 지도 시각화 유형

| # | 유형 | 한국어명 | 영문명 | 설명 |
|---|------|---------|--------|------|
| 9 | 단계구분도 | 단계구분도 | Choropleth Map | 행정구역을 등급별 색상으로 채워 분포를 표현 |
| 10 | 등치선도 | 등치선도 | Isoline Map (Isarithmic) | 동일 값을 연결하는 등치선을 지도 위에 표현 |
| 11 | 도형표현도 | 도형표현도 | Proportional Symbol Map | 각 지역 위에 파이 차트 또는 막대 차트를 배치 |

### 1.3 핵심 가치
- **원스톱 도구**: 그래프 + 지도 시각화를 한 곳에서 제작
- **직관적 조작**: 클릭/드래그로 색칠, 데이터 입력으로 자동 생성
- **선명한 출력**: 범례 포함 고해상도 PNG (3× 레티나 대응)
- **수업 최적화**: 흑백 인쇄 대응, 교과서 스타일 프리셋
- **한국어 전용 UI**: 기존 GeoGrapher와 동일

### 1.4 기술 스택 (기존 GeoGrapher 계승)
- React 18 + TypeScript + Vite
- 기존 차트: Canvas 2D 렌더링
- **지도 모듈: SVG 렌더링** (D3.js v7 — geo 투영, 색상 스케일)
- 배포: Vercel (geographer.bgnl.kr)

## 3. 공통 기능 — 3가지 지도 유형 모두 적용

### 3.1 행정구역 단위 선택

| 모드 | 명칭 | 구성 | 피처 수 (약) | 설명 |
|------|------|------|-------------|------|
| A | **시·도** | 17개 광역 행정구역 | 17 | 서울, 부산, …, 제주 |
| B | **시·군·구** | 전국 시·군·구 전체 | ~250 | 모든 기초 자치단체 + 자치구 |
| C | **시·군·구 (광역 통합)** | 도 소속 시·군 + 광역시/특별시 병합 | ~170 | 광역시/특별시/특별자치시는 하나로 묶고, 도는 시·군·구 단위 |

- 단위 전환 시 **현재 데이터/색칠 초기화** (확인 모달)
- 모드 C 광역 통합: 자치구 폴리곤을 dissolve하여 단일 폴리곤으로 사전 빌드

### 3.2 SVG 렌더링 공통

| 항목 | 상세 |
|------|------|
| 렌더링 | SVG 기반 (D3.js v7 geoPath) |
| 투영법 | Mercator (기본) — 한반도 자동 fitSize |
| 확대/축소 | d3-zoom (마우스 휠 + 드래그 패닝) |
| 기본 스타일 | 흰색 배경, 미지정 지역 #f0f0f0 |
| 경계선 | 시·도: 1.5px #888 / 시·군·구: 0.5px #ccc |
| 호버 효과 | 지역 위 마우스 오버 시 테두리 강조 + 툴팁(지역명, 값) |

### 3.3 지역 라벨

- ON/OFF 토글 (시·군·구 모드 기본 OFF)
- 약칭/전체 명칭 모드
- 폰트 크기 조절 (소/중/대)
- 라벨 색상: 배경 명도 기반 자동 흑/백 전환
- 시·군·구 모드: 줌 레벨에 따라 동적 표시
- 호버 툴팁은 라벨 OFF에서도 항상 동작

### 3.4 패턴 시스템 (Fill Pattern)

기존 GeoGrapher 누적 막대/원 그래프에서 사용하는 **grayscale + pattern** 세트를 지도 모듈에서도 동일하게 사용한다. 색상만으로 구분이 어려운 흑백 인쇄 환경에서 필수적이며, 컬러 모드에서도 패턴을 중첩하여 가독성을 높일 수 있다.

#### 3.4.1 적용 대상

| 지도 유형 | 패턴 적용 영역 | 설명 |
|----------|--------------|------|
| 단계구분도 | 행정구역 폴리곤 fill | 등급별로 서로 다른 패턴 적용 |
| 도형표현도 — 파이 | 파이 세그먼트 fill | 항목별로 서로 다른 패턴 적용 |
| 도형표현도 — 막대 | 막대 rect fill | 항목별로 서로 다른 패턴 적용 |
| 등치선도 | ❌ 미적용 | 선 기반 표현이므로 패턴 불필요 |

#### 3.4.2 패턴 세트 (기존 GeoGrapher와 동일)

기존 Canvas 패턴을 SVG `<pattern>` 요소로 1:1 변환한다.

| # | 패턴명 | 형태 | SVG 구현 |
|---|--------|------|----------|
| 1 | 사선 (\\\\) | 좌하→우상 45° 평행선 | `<line>` 반복, `patternTransform="rotate(45)"` |
| 2 | 역사선 (//) | 우하→좌상 135° 평행선 | `<line>` 반복, `patternTransform="rotate(-45)"` |
| 3 | 가로선 (──) | 수평 평행선 | `<line>` 반복 |
| 4 | 세로선 (││) | 수직 평행선 | `<line>` 반복 |
| 5 | 격자 (╬) | 가로 + 세로 교차 | `<line>` × 2 |
| 6 | 대각 격자 (╳) | 45° + 135° 교차 | `<line>` × 2 |
| 7 | 점 (· · ·) | 균일 분포 원형 점 | `<circle>` 반복 |
| 8 | 큰 점 (● ● ●) | 큰 원형 점 | `<circle>` 반복 (반지름 ↑) |

- 패턴 선 굵기: 기본 1.5px (조절 가능)
- 패턴 간격: 기본 8px (조절 가능: 4/6/8/10/12px)
- 패턴 색상: 기본 검정 (#333) — 배경색과의 대비를 위해 자동 조절 가능

#### 3.4.3 채우기 모드 (Fill Mode) 전환

도구 패널에 **채우기 모드** 토글을 제공한다.

| 모드 | 설명 | fill 값 |
|------|------|---------|
| **컬러** (기본) | 색상만으로 채움 | `fill="색상"` |
| **패턴** | 패턴만으로 채움 (배경 흰색/회색) | `fill="url(#pattern-N)"` |
| **컬러 + 패턴** | 색상 위에 패턴 중첩 | `fill="색상"` + 패턴 오버레이 `<rect fill="url(#pattern-N)">` |
| **그레이스케일 + 패턴** | 명도 단계 + 패턴 중첩 | `fill="gray-N"` + 패턴 오버레이 |

#### 3.4.4 SVG 패턴 구현

```svg
<defs>
  <!-- 패턴 1: 사선 -->
  <pattern id="pattern-diagonal" width="8" height="8"
           patternUnits="userSpaceOnUse"
           patternTransform="rotate(45)">
    <line x1="0" y1="0" x2="0" y2="8"
          stroke="#333" stroke-width="1.5" />
  </pattern>

  <!-- 패턴 7: 점 -->
  <pattern id="pattern-dots" width="8" height="8"
           patternUnits="userSpaceOnUse">
    <circle cx="4" cy="4" r="1.5" fill="#333" />
  </pattern>

  <!-- 컬러+패턴 중첩 시: 패턴에 반투명 적용 -->
  <pattern id="pattern-diagonal-overlay" width="8" height="8"
           patternUnits="userSpaceOnUse"
           patternTransform="rotate(45)">
    <line x1="0" y1="0" x2="0" y2="8"
          stroke="rgba(0,0,0,0.5)" stroke-width="1.5" />
  </pattern>
</defs>
```

- `<defs>`는 SVG 최상위에 한 번만 정의, 모든 지도 유형에서 `url(#pattern-id)`로 참조
- 컬러+패턴 중첩 시: 폴리곤/세그먼트 위에 동일 shape의 `<path>` 또는 `<rect>`를 하나 더 쌓고 `fill="url(#pattern-overlay)"`
- PNG 내보내기 시 패턴도 SVG 내부에 포함되므로 별도 처리 불필요

#### 3.4.5 패턴 선택 UI

```
┌─ 채우기 모드 ─────────────────┐
│  ○ 컬러  ○ 패턴              │
│  ○ 컬러+패턴  ○ 그레이+패턴   │
├───────────────────────────────┤
│  등급 1:  [색상■] [패턴 ▾ \\]  │
│  등급 2:  [색상■] [패턴 ▾ //]  │
│  등급 3:  [색상■] [패턴 ▾ ──]  │
│  등급 4:  [색상■] [패턴 ▾ ││]  │
│  등급 5:  [색상■] [패턴 ▾ ··]  │
└───────────────────────────────┘
```

- 패턴 프리뷰: 드롭다운 내에서 실제 패턴 모양을 미리보기

- 제목·단위·출처 텍스트 직접 입력
- 지도 유형별 범례 내용은 각 섹션에서 별도 기술

### 3.6 데이터 입력 방식

| 방식 | 단계구분도 | 등치선도 | 도형표현도 | 설명 |
|------|-----------|---------|-----------|------|
| 클릭/드래그 색칠 | ✅ | ❌ | ❌ | 팔레트 선택 후 지역 클릭 |
| 직접 입력 테이블 | ✅ | ✅ | ✅ | 지역별 수치를 테이블에 입력 |
| CSV 업로드 | ✅ | ✅ | ✅ | CSV 파일 업로드 → 자동 매핑 |
| Google Sheets 연동 | ✅ | ✅ | ✅ | 기존 GeoGrapher의 Sheets 임포트 재활용 |

### 3.7 PNG 내보내기

| 항목 | 기본값 | 옵션 |
|------|--------|------|
| 해상도 배율 | 3× | 1×, 2×, 3× |
| 배경 | 흰색 | 흰색, 투명 |
| 범례 포함 | ON | ON/OFF |
| 지역 라벨 포함 | 현재 설정 | ON/OFF |
| 여백(padding) | 20px | 0/10/20/40px |

기술 구현:
```
SVG DOM (지도 + 범례 + 라벨 + 심볼 모두 포함)
  ↓  웹폰트 인라인 (Pretendard Base64)
  ↓  CSS → inline style 변환
  ↓  SVG serialize
  ↓  Canvas drawImage (N× 해상도)
  ↓  canvas.toBlob('image/png')
  ↓  다운로드: GeoGrapher_단계구분도_YYYYMMDD_HHmmss.png
```

### 3.8 프로젝트 저장/불러오기

- 현재 상태를 JSON으로 내보내기/불러오기
- 저장 항목: 지도 유형, 행정구역 단위, 데이터, 색상, **채우기 모드, 등급/항목별 패턴 종류**, 범례 설정(위치·제목·단위·출처)
- 파일명: `GeoGrapher_[유형]_[날짜].json`

## 4. 유형별 세부 기능

### 4.1 단계구분도 (Choropleth Map)

#### 4.1.1 개념
행정구역 폴리곤 전체를 등급별 색상으로 채워, 공간적 분포·크기·밀도를 시각화한다.

#### 4.1.2 색칠 모드

**모드 A — 수동 색칠 (기본)**
- 좌측 팔레트에서 등급(색상) 선택 → 지역 클릭 → fill 적용
- 드래그 모드 토글: 마우스를 누른 채 여러 지역 연속 색칠
- 우클릭: 해당 지역 색상 초기화
- 같은 색상 재클릭: 토글(해제)

**모드 B — 데이터 자동 분류**
- 데이터 입력 테이블 또는 CSV로 지역별 수치 입력
- 등급 분류 방식 선택:
  - 등간격 (Equal Interval)
  - 분위수 (Quantile)
  - 자연 분류 (Jenks Natural Breaks)
  - 수동 경계값 입력
- 단계 수: 3~7 (기본 5)
- 분류 결과 자동 색칠 + 범례 자동 생성

#### 4.1.3 색상 팔레트 및 패턴

| 팔레트 | 설명 | 예시 용도 |
|--------|------|----------|
| 순차형 (Sequential) | 연한색 → 진한색 단일 색조 | 인구밀도, 생산량 |
| 발산형 (Diverging) | 양극 (파랑↔빨강) | 증감률, 찬반 비율 |
| 범주형 (Categorical) | 서로 다른 색상 | 권역 분류, 작물 종류 |
| 흑백 명도형 | 명도 5~7단계 | 흑백 인쇄 대응 |
| 커스텀 | 각 단계별 Color Picker | 자유 지정 |

**패턴 적용** (§3.4 참조)
- 행정구역 폴리곤에 패턴을 적용하여 흑백 인쇄 시에도 등급 구분 가능
- 채우기 모드 4종: 컬러 / 패턴 / 컬러+패턴 / 그레이스케일+패턴
- 등급별 패턴 종류 개별 지정 가능 (사선, 역사선, 점 등 8종 — 기존 누적 막대/원 그래프와 동일 세트)
- 수동 색칠 모드에서: 팔레트의 각 등급에 색상+패턴이 함께 표시됨
- 데이터 자동 분류 모드에서: 등급별 패턴 자동 배정 (순서대로)

#### 4.1.4 범례 내용

컬러 모드:
```
┌────────────────────┐
│  인구밀도 (명/km²)   │
│                    │
│  ■ 1,000 이상       │
│  ■ 500 ~ 999       │
│  ■ 200 ~ 499       │
│  ■ 100 ~ 199       │
│  ■ 100 미만         │
│                    │
│  출처: 통계청, 2024  │
└────────────────────┘
```

패턴 모드 / 그레이+패턴 모드:
```
┌────────────────────┐
│  인구밀도 (명/km²)   │
│                    │
│  [\\\\] 1,000 이상   │
│  [////] 500 ~ 999   │
│  [────] 200 ~ 499   │
│  [····] 100 ~ 199   │
│  [    ] 100 미만     │
│                    │
│  출처: 통계청, 2024  │
└────────────────────┘
```
- 범례 스와치에 실제 SVG 패턴을 렌더링하여 인쇄 시에도 정확히 대응

### 4.2 등치선도 (Isoline Map)

#### 4.2.1 개념
동일한 값을 가진 지점들을 연결하는 **등치선(iso-contour)**을 지도 위에 표시한다. 등고선, 등온선, 등압선, 등우선, 등심선 등에 해당한다.

#### 4.2.2 데이터 입력

**포인트 기반 입력** — 지역(행정구역 중심점 또는 사용자 지정 좌표)별 수치를 입력하면 보간(interpolation)하여 등치선을 생성한다.

| 입력 방식 | 설명 |
|----------|------|
| 행정구역 기반 | 시·도 또는 시·군·구 선택 → 중심점(centroid)에 값 부여 → 보간 |
| 자유 포인트 | 지도 위 클릭으로 좌표 지정 + 값 입력 (관측소 위치 등) |
| CSV 업로드 | `위도, 경도, 값` 또는 `행정구역명, 값` 형식 |
| 직접 입력 테이블 | 좌측 패널에서 지역별 수치 입력 |

#### 4.2.3 보간 및 등치선 생성

```
포인트 데이터 (x, y, value)
  ↓
공간 보간 (Interpolation)
  - IDW (Inverse Distance Weighting, 기본)
  - 또는 단순 선형 보간
  ↓
그리드 래스터 생성 (해상도 조절 가능)
  ↓
등치선 추출 (d3-contour / Marching Squares)
  ↓
SVG <path> 렌더링 (행정구역 경계에 clip)
```

#### 4.2.4 등치선 스타일 설정

| 설정 항목 | 기본값 | 옵션 |
|----------|--------|------|
| 등치선 간격 | 자동 (데이터 범위 ÷ 단계 수) | 수동 입력 |
| 단계 수 | 5 | 3~10 |
| 선 색상 | 검정 (#333) | Color Picker |
| 선 굵기 | 주곡선 1.5px / 간곡선 0.7px | 조절 가능 |
| 선 스타일 | 실선 | 실선, 파선 |
| 등치선 값 표시 | ON | ON/OFF — 선 위에 수치 라벨 |
| 색상 채우기 | OFF | ON 시 등치선 사이를 색상 그라데이션으로 채움 |
| 클리핑 | 행정구역 경계 내부만 | ON/OFF |

#### 4.2.5 주곡선 / 간곡선

- 주곡선: 기준 간격마다 (예: 100mm 간격) — 굵은 선 + 값 라벨
- 간곡선: 주곡선 사이 세분 (예: 50mm 간격) — 가는 선
- 사용자가 주곡선 간격과 간곡선 분할 수를 설정

#### 4.2.6 범례 내용
```
┌───────────────────────┐
│  연 강수량 (mm)        │
│                       │
│  ── 1,600             │
│  ── 1,400             │
│  ── 1,200             │
│  ── 1,000             │
│                       │
│  ── 주곡선 (200mm 간격) │
│  -- 간곡선 (100mm 간격) │
│                       │
│  출처: 기상청, 2024     │
└───────────────────────┘
```
색상 채우기 ON 시:
```
┌───────────────────────┐
│  연 강수량 (mm)        │
│                       │
│  ■ 1,600 이상          │
│  ■ 1,400 ~ 1,600      │
│  ■ 1,200 ~ 1,400      │
│  ■ 1,000 ~ 1,200      │
│  ■ 1,000 미만          │
└───────────────────────┘
```

#### 4.2.7 수업 활용 예시
- 연 강수량 등우선도 (기상청 관측소 데이터)

### 4.3 도형표현도 (Proportional Symbol Map)

#### 4.3.1 개념
각 행정구역의 중심점(centroid) 위에 **파이 차트** 또는 **막대 차트**를 배치하여, 지역별 구성비·항목별 크기를 동시에 표현한다.

#### 4.3.2 도형 유형

##### A. 파이 차트 (Pie Chart Symbol)

각 지역에 원형 파이 차트를 배치한다.

```
        ┌───┐
       /  A  \        A: 1차 산업 (30%)
      │   /───│       B: 2차 산업 (25%)
      │  / B  │       C: 3차 산업 (45%)
       \ C  /
        └───┘
      [서울]
```

| 설정 항목 | 기본값 | 옵션 |
|----------|--------|------|
| 항목 수 | 3 | 2~8 |
| 각 항목 색상 | 범주형 팔레트 | Color Picker |
| 각 항목 패턴 | 없음 (컬러 모드) | 8종 패턴 중 선택 (§3.4) |
| 채우기 모드 | 컬러 | 컬러 / 패턴 / 컬러+패턴 / 그레이+패턴 |
| 원 크기 모드 | 균일 (고정 반지름) | 비례 (총합에 비례) |
| 고정 반지름 | 20px | 10~50px 슬라이더 |
| 비례 시 최소/최대 반지름 | 10px / 40px | 조절 가능 |
| 테두리 | 1px 흰색 | 색상/두께 조절 |
| 값 표시 | OFF | 비율(%) / 절대값 / OFF |

##### B. 막대 차트 (Bar Chart Symbol)

각 지역에 수직 막대 차트를 배치한다.

```
      █ █
      █ █ █
      █ █ █ █
      ─────────
      A B C D
      [서울]
```

| 설정 항목 | 기본값 | 옵션 |
|----------|--------|------|
| 항목 수 | 3 | 2~6 |
| 각 항목 색상 | 범주형 팔레트 | Color Picker |
| 각 항목 패턴 | 없음 (컬러 모드) | 8종 패턴 중 선택 (§3.4) |
| 채우기 모드 | 컬러 | 컬러 / 패턴 / 컬러+패턴 / 그레이+패턴 |
| 막대 크기 모드 | 절대값 비례 | 비율 기준(100% 고정 높이) |
| 최대 높이 | 40px | 20~80px 슬라이더 |
| 막대 너비 | 6px | 4~12px |
| 막대 간격 | 1px | 0~4px |
| 값 표시 | OFF | 수치 / OFF |
| 기준선 | 하단 가로선 | ON/OFF |

#### 4.3.3 데이터 입력

**데이터 테이블 구조 (파이/막대 공통)**

| 지역 | 항목1 | 항목2 | 항목3 | ... |
|------|-------|-------|-------|-----|
| 서울 | 120 | 340 | 890 | ... |
| 부산 | 80 | 250 | 620 | ... |
| ... | ... | ... | ... | ... |

- 지역명은 GeoJSON의 `name` 또는 `name_short`와 자동 매칭
- 항목 헤더: 사용자가 직접 입력 (예: "1차 산업", "2차 산업", "3차 산업")
- 비율 모드: 입력값을 각 지역의 합계 기준 비율로 자동 변환
- CSV 업로드 지원

#### 4.3.4 심볼 배치

- 기본 위치: 각 행정구역 폴리곤의 centroid (d3 centroid)
- **수동 위치 조정**: 심볼을 드래그하여 위치 미세 조정 가능 (좁은 행정구역에서 겹침 해결)
- 시·군·구 모드에서는 심볼 크기 자동 축소 또는 줌 레벨에 따른 가시성 제어
- 심볼 겹침 감지: 겹치는 경우 경고 표시

파이 차트 — 컬러 모드:

```

│  [····] 3차 산업       │
│                      │
│  원 크기: 총 생산액    │

#### 4.3.6 수업 활용 예시
- 시·도별 산업 구조 비율 (파이)
- 시·도별 에너지원별 발전량 (막대)

  ├─ 서울특별시 용산구  ─┤
  ├─ ...               ─┘
  ├─ 경기도 수원시 장안구 → 그대로 유지
  └─ ...
```

- 병합 대상: 행정구역 코드 앞 2자리가 `11`(서울), `21`(부산), `22`(대구), `23`(인천), `24`(광주), `25`(대전), `26`(울산), `29`(세종)
- 빌드 타임에 Python(geopandas) 또는 mapshaper CLI로 사전 생성

### 5.4 Properties 표준 필드

| 필드명 | 타입 | 설명 | 예시 |
|--------|------|------|------|
| `code` | string | 행정구역 코드 | `"11"`, `"11010"` |
| `name` | string | 전체 명칭 | `"서울특별시"`, `"수원시 장안구"` |
| `name_short` | string | 약칭 | `"서울"`, `"수원 장안"` |
| `parent_code` | string | 상위 시·도 코드 (2자리) | `"11"`, `"31"` |
| `parent_name` | string | 상위 시·도 명칭 | `"서울특별시"`, `"경기도"` |
| `region_type` | string | 구분 | `"sido"` / `"sigungu"` / `"metro_merged"` |

### 5.5 파일 크기 최적화
- mapshaper `-simplify dp 10%`로 꼭짓점 감소
- TopoJSON 변환 고려 (공유 경계 중복 제거 최대 80%)
- GeoJSON 파일은 `public/geo/`에 배치, Vite 빌드 시 그대로 복사

## 7. 세부 동작 명세

### 7.1 GeoJSON 로드 흐름

```
차트 유형에서 지도 시각화 선택
  ↓
행정구역 단위 드롭다운 확인 (기본: 시·도)
  ↓
useGeoData 훅 → fetch(`/geo/sido.geojson`)
  ↓  (캐시 히트 시 생략)
D3 geoPath + projection 설정
  ↓
SVG <path> 렌더링
  ↓
인터랙션 레이어 활성화
```

### 7.2 등치선 생성 파이프라인 (등치선도)

```
포인트 데이터 [{lat, lng, value}, ...]
  ↓
d3.geoProjection으로 화면 좌표 변환
  ↓
IDW 보간 → 래스터 그리드 (200×200 기본)
  ↓
d3.contours() → 등치선 GeoJSON 배열
  ↓
행정구역 폴리곤으로 clipPath 적용
  ↓
SVG <path> 렌더링 (stroke + optional fill)
```

### 7.3 도형표현도 심볼 렌더링 (도형표현도)

```
데이터 테이블 {지역코드: {항목1: v1, 항목2: v2, ...}, ...}
  ↓
각 지역의 centroid 계산 (d3 path.centroid)
  ↓
[파이 모드]
  SVG <g> 안에 d3.arc()로 파이 세그먼트 렌더링
  크기 = 총합 비례 또는 고정
  채우기 모드에 따라:
    컬러 → fill="색상"
    패턴 → fill="url(#pattern-N)"
    컬러+패턴 → fill="색상" + 동일 arc의 패턴 오버레이 <path>
    그레이+패턴 → fill="gray-N" + 패턴 오버레이

[막대 모드]
  SVG <g> 안에 <rect> 배열로 막대 렌더링
  높이 = 값 비례, 너비 = 고정
  채우기 모드에 따라:
    컬러 → fill="색상"
    패턴 → fill="url(#pattern-N)"
    컬러+패턴 → fill="색상" + 동일 rect 위에 패턴 오버레이 <rect>
    그레이+패턴 → fill="gray-N" + 패턴 오버레이
  ↓
심볼 위치 드래그 가능 (겹침 해결)
```

### 7.4 한글 폰트 PNG 렌더링 보장
- Pretendard 웹폰트 (CDN)
- SVG → Canvas 변환 시 폰트 Base64 인라인 삽입
- `document.fonts.ready` 이후 내보내기 실행
- `<text>` 요소만 사용 (`foreignObject` 미사용)

## 8. 개발 우선순위

8. 단계구분도 드래그 색칠 모드
9. 단계구분도 데이터 입력 → 자동 등급 분류
10. **SVG PatternDefs 구현 (8종 패턴 — 기존 Canvas 패턴 1:1 변환)**
11. **채우기 모드 UI (컬러/패턴/컬러+패턴/그레이+패턴)**
12. **단계구분도 패턴 적용 + 범례 패턴 스와치**
13. 도형표현도 — 파이 차트 모드
14. 도형표현도 — 막대 차트 모드
15. **도형표현도 패턴 적용 (파이 세그먼트 + 막대 rect)**
16. 도형표현도 데이터 입력 테이블
17. 심볼 드래그 위치 조정
18. 범례 드래그 배치 + 제목/단위/출처 편집

### Phase 3: 등치선도 + 고급 기능
19. 등치선도 포인트 데이터 입력
20. IDW 보간 + d3-contour 등치선 생성
21. 주곡선/간곡선 설정
22. 등치선 색상 채우기 모드
23. 다양한 팔레트 프리셋 (발산형, 범주형, 흑백)
24. 지역 라벨 (줌 레벨 연동)
25. CSV / Google Sheets 임포트 (기존 GeoGrapher 파서 재활용)
26. 프로젝트 JSON 저장/불러오기

### Phase 4: 폴리싱
27. 다크모드 (기존 GeoGrapher 테마 연동)
28. 반응형 모바일 대응
29. 성능 최적화 (시·군·구 ~250개 심볼 렌더링)
30. 접근성 (키보드 내비게이션, ARIA)
